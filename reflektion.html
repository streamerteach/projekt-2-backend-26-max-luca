Reflektion – Projekt 2 (PHP och SQL)

I detta projekt har vi lärt oss mycket om hur databaser fungerar tillsammans med PHP. Fokus låg inte bara på att bygga sidan, utan framför allt på hur man sparar, hämtar och uppdaterar data i en SQL-databas.

Det som var lättast i början var att göra enkla SELECT och INSERT. Att hämta data och skriva ut det på sidan gick ganska snabbt att förstå. När vi väl fattade hur en tabell är uppbyggd (kolumner och rader) blev det också lättare att jobba vidare.

Det som var svårare var UPDATE och DELETE tillsammans med WHERE. Små misstag där kunde göra att fel data ändrades eller att inget hände alls. Vi spenderade ganska mycket tid på att förstå exakt vilken data som påverkas av en query.

En annan svår del var att koppla ihop databasen med PHP. Till exempel att använda prepare(), execute() och fetch() rätt. I början var det förvirrande varför vissa saker inte fungerade, fast koden såg rätt ut.

Vi tyckte också att sessions och login var lite svåra i början. Att förstå hur användaren "hålls inloggad" och hur man använder $_SESSION tog lite tid. Samma sak med lösenordshashning, men efter att vi testade det flera gånger blev det tydligare.

Felsökning tog mest tid i hela projektet. Ofta var det små fel som saknade variabler, fel fil som inte var inkluderad, eller problem med databaskopplingen. Vi lärde oss att läsa felmeddelanden noggrant och testa steg för steg.

Det som var roligast var när databasen började fungera på riktigt. Till exempel när vi kunde uppdatera profiler, spara data och se ändringarna direkt. Då kändes det som att allt hängde ihop.

Om vi skulle göra projektet igen skulle vi planera databasen bättre från början. Till exempel tänka igenom alla kolumner och datatyper innan vi börjar koda. Det hade sparat tid senare.

Vi skulle också strukturera koden bättre direkt, så att vi slipper ändra så mycket i efterhand.

Sammanfattningsvis har projektet gett oss en mycket bättre förståelse för databaser och hur PHP kommunicerar med SQL. Det var ibland svårt, men vi lärde oss mycket genom att testa och lösa problem själva.
